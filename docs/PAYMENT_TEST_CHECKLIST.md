# Payment Test Checklist

## 상품 설정
- courseId: `dui-prevention-basic`
- 상품명: `음주운전 예방교육`
- 결제금액: 상품별 실제 결제금액
- 총 강의 수: `5강`
- 수강기간: 결제일로부터 `90일`
- 환불 산정 기준: `실제 결제금액 × 미수강 강의 수 / 전체 강의 수`

## 결제 페이지
- 환불규정 동의 체크 전에는 결제 버튼 비활성화
- 환불규정 동의 체크 후 결제 버튼 활성화
- 결제 요청 금액이 `59,000원`인지 확인
- 결제 상품명이 `음주운전 예방교육`인지 확인
- 결제 페이지에 미수강 강의별 환불 가능 금액표 표시 확인

## 결제 성공
- successUrl에서 `paymentKey`, `orderId`, `amount`, `courseId` 수신
- Worker `/api/payments/confirm` 호출
- 서버에서 courseId 기준 금액 `59,000원` 재검증
- `payments` 컬렉션 결제내역 저장
- `enrollments` 컬렉션 수강권 저장
- 기존 호환용 `purchases` 컬렉션 저장
- 결제 후 즉시 강의 접근 가능

## 수강기간 및 환불
- 결제일로부터 90일 후 강의 접근 불가
- 만료 안내 문구: `해당 강의의 수강기간은 결제일로부터 90일이며, 현재 수강기간이 만료되어 수강할 수 없습니다.`
- 0강 수강 시 결제금액의 5/5 환불 가능
- 1강 수강 시 결제금액의 4/5 환불 가능
- 2강 수강 시 결제금액의 3/5 환불 가능
- 3강 수강 시 결제금액의 2/5 환불 가능
- 4강 수강 시 결제금액의 1/5 환불 가능
- 5강 수강 시 환불 불가
- 수료증 발급 후 환불 불가

## 보안
- 프론트 금액 위변조 시 Worker 승인 거부
- 동일 `orderId` 중복 요청 시 거부
- 동일 `paymentKey` 중복 요청 시 거부
- `PAYMENT_SECRET_KEY`는 Worker secret으로만 설정
- Firebase ID 토큰 uid와 요청 uid 불일치 시 거부
- DB 저장 실패 시 `paymentLogs`에 재처리 로그 저장
- 이미 활성 수강권이 있는 동일 강의 중복 결제 거부

## 실패 페이지
- failUrl에서 `code`, `message` 표시
- 주문서 재시도 버튼 표시
