# Firebase Beginner Prompt

아래 프롬프트를 AI(ChatGPT, Claude, Cursor 등)에 그대로 복사해서 붙여넣으세요.

```md
나는 개발 초보자다.
한 번에 모든 내용을 설명하지 말고, 반드시 1단계씩 아주 천천히 진행해줘.

중요한 규칙:
1. 절대 여러 단계를 한 번에 설명하지 마.
2. 매 단계마다 내가 해야 할 행동을 아주 구체적으로 알려줘.
3. 어려운 용어를 쓰면 바로 아래에 초보자도 이해할 수 있게 쉬운 말로 풀이해줘.
4. 각 단계 설명은 짧고 명확하게 해줘.
5. 각 단계가 끝나면 반드시 내가 확인할 수 있는 체크포인트를 알려줘.
6. 내가 "다음 단계"라고 말하기 전에는 절대 다음 단계로 넘어가지 마.
7. 내가 에러 메시지를 보내면, 그 단계에서 왜 에러가 났는지 초보자 눈높이로 설명하고 해결 순서만 다시 안내해줘.
8. 설명보다 실습 중심으로 진행해줘.
9. 내가 복사해서 붙여넣을 코드가 있다면, 어디 파일에 넣어야 하는지 먼저 설명하고 코드를 보여줘.
10. 백엔드는 Firebase 기준으로만 설명해줘. 다른 백엔드 서비스는 제안하지 마.

내 목표:
- Firebase를 백엔드로 사용하는 웹 서비스를 처음부터 차근차근 만들고 싶다.
- 나는 백엔드 경험이 거의 없다.
- 복잡한 구조 설명보다, 지금 당장 해야 하는 한 가지 작업이 더 중요하다.
- 각 단계마다 "왜 이 작업을 하는지"도 한두 문장으로 설명해줘.

진행 방식:
- 항상 아래 형식으로 답변해줘.

[현재 단계]
이번에 할 일 1가지만 설명

[왜 하는지]
초보자용 쉬운 설명

[실행 방법]
1. 어디로 들어간다
2. 무엇을 클릭한다
3. 어떤 코드를 붙여넣는다
4. 무엇을 확인한다

[체크포인트]
정상이라면 무엇이 보여야 하는지

[다음으로 넘어가는 방법]
내가 AI에게 뭐라고 말하면 되는지 예: "다음 단계"

내 프로젝트 조건:
- 백엔드는 Firebase로 고정
- 가능한 한 초보자가 가장 덜 헷갈리는 방식으로 설명
- 필요하면 Firebase 콘솔에서 클릭할 메뉴 이름까지 자세히 안내
- 필요하면 로컬 프로젝트 폴더 구조도 같이 설명

지금 바로 시작하지 말고,
먼저 내가 만들려는 서비스가 무엇인지 3문장 이내로 물어봐줘.
그리고 내 답변을 받은 뒤에만 1단계부터 시작해줘.
```

추천 사용법:

1. 위 프롬프트를 AI에 붙여넣습니다.
2. AI가 서비스 아이디어를 물어보면 짧게 답합니다.
3. 이후에는 `다음 단계`라고 입력하면서 한 단계씩 진행합니다.
4. 오류가 나면 에러 화면이나 메시지를 그대로 붙여넣습니다.

추가 팁:

- AI가 한 번에 너무 많이 설명하면 `한 단계만 다시 설명해줘`라고 말하세요.
- 코드가 길면 `이 코드를 어느 파일에 넣는지 먼저 설명해줘`라고 말하세요.
- Firebase 콘솔 작업이 헷갈리면 `클릭 순서를 더 자세히 알려줘`라고 말하세요.
